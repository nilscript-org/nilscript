# pocketbase-nil-adapter (in-core canonical example)

> 🟢 **Also published standalone as the Official Verified Adapter:**
> [nilscript-org/pocketbase-nil-adapter](https://github.com/nilscript-org/pocketbase-nil-adapter).
> This copy stays *in* the core as the canonical example that is regenerated and re-verified
> against the current `scaffold-shim` templates in the same commit (instant-consistency proof).
> For installing/using the adapter, prefer the standalone repo.

A NIL translation shim for **pocketbase**, generated by `nilscript scaffold-shim`.

The NIL edge (`src/pocketbase_nil_adapter/edge.py`) is generic and complete. You fill **two** files:

- `src/pocketbase_nil_adapter/translate.py` — map each active NIL verb to a native document (and back).
- `src/pocketbase_nil_adapter/system.py` — `RealSystemClient`: the only I/O against pocketbase.

## Discover the hidden requirements (do this once)

```
nilscript scan --url https://your-pocketbase.example --safe   # -> requirements-manifest.json
```

`scan` learns the requirements your backend does not advertise (required fields, prerequisite
entities, transport quirks) so you do not re-learn them by collision. The shim reads that manifest
and pre-fills them automatically (`src/pocketbase_nil_adapter/manifest.py`).

## Verify

```
pip install -e ".[dev]"
pytest                         # conformance proof against the in-memory FakeSystem
```

The generated stubs raise `NotImplementedError` — until you fill `translate.py`, conformance fails
every active verb (by design: it proves the harness detects non-conformance, not just conformance).
